About me

Filippo

21 anni
Perugia
La curiosità è il mio linguaggio preferito!

C'è stato un tempo in cui la parola «developer» non significava niente per me. La incontrai per caso in una community online, accanto al nome di qualcuno come fosse un titolo nobiliare. «E quello cosa fa?». Ricevetti una risposta che, lo ammetto, non chiariva granché, ma bastò a farmi cadere nella tana del coniglio. Da lì non sono più uscito: oggi quel termine misterioso è semplicemente il mio mestiere. Mi chiamo Filippo, e uno sviluppatore Java e Web.

Ciò che mi appassiona non è soltanto scrivere codice, ma veder nascere qualcosa di concreto da un'idea: prendere un problema, scomporlo e osservare i pezzi incastrarsi finché tutto non funziona. Sono un curioso per natura: mi affascina l'universo e in generale tutto ciò che mi spinge a chiedermi come funzionano le cose; in fondo, la stessa domanda che mi tiene davanti allo schermo.

Mi considero uno sviluppatore Mid-Level (intermedio): gran parte di ciò che so l'ho imparato da autodidatta, costruendo progetti reali e sbagliando finché non funzionavano. Oggi sto consolidando queste competenze con un corso ITS Umbria in ambito IT che, insieme alle prime esperienze lavorative, mi sta aiutando a muovermi nel mondo del lavoro come developer, oltre a dare forma ai progetti personali che mostro di seguito.

Java

Esperienza dal 2021

Il mio linguaggio principale. Lavoro da anni su plugin Bukkit/Spigot per Minecraft e su bot Discord personalizzati costruiti con la libreria JDA.

IDE & Editor

Uso professionale quotidiano

Abbonato a tutti i servizi JetBrains, anche se uso attivamente solo quelli che mi servono davvero.

  • IntelliJ IDEA Ultimate — il mio IDE principale per Java
  • PhpStorm — per i progetti web/PHP
  • VS Code — editor leggero per modifiche rapide, ad es. file di configurazione

Database

Buona, non da esperto

Lavoro principalmente con MySQL e SQLite, quest'ultimo il più usato. Conosco le query essenziali necessarie nei progetti tipo plugin Bukkit, non l'intero linguaggio SQL.

  • MySQL e SQLite — uso quotidiano
  • SQL Server — conoscenza di base

Docker

Esperienza pratica

Per lo sviluppo quotidiano preferisco un ambiente Ubuntu via WSL, ma so creare e gestire container Docker quando un progetto richiede librerie o versioni software diverse da quelle già installate sul sistema.

Git & Version control

Uso quotidiano

Uso Git quotidianamente in tutti i miei progetti. Il mio GitHub pubblico fornito contiene principalmente progetti per divertimento e portfolio, ogni cosa che vedi è intenzionalmente pubblica per mostrare il mio work style.

I veri progetti su cui lavoro sono spesso under NDA (non-disclosure agreement), quindi ospitati su repository private. Ho veramente tanti progetti sviluppati.
